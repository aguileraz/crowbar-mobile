# Coding Standards for Python

## Code Style
- Follow the [PEP 8](https://peps.python.org/pep-0008/) standard
- Use `black` for automatic formatting
- Use `isort` for organizing imports

## Project Structure
- Separate code into logical modules within `src/`
- Use `__init__.py` to make directories Python packages

## Best Practices
- Prefer composition over inheritance
- Use type hints with `typing` whenever possible
- Avoid side effects in modules
